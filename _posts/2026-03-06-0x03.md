# 배열이란
메모리 상에 원소를 연속하게 배치한 자료구조

## 성질
추가적으로 소모되는 메모리 양이 거의 없음
Cache hit rate가 높음
메모리 상에 연속한 구간을 잡아야 해서 할당에 제약이 걸림

## 기능의 시간 복잡도
O(1)
1. K번쨰 원소 확인/변경
2. 원소를 마지막에 추가
3. 마지막 원소 제거

O(N)
1. K번쨰에 원소 추가/제거

# 배열과 캐시
배열에 관련된 작업을 수행하기 위해서는 배열의 정보를 캐시에 올려야한다.
이떄 배열은 메모리 상에서 연속적으로 존재하기에 다음 번 연산에서 필요한 정보가 이미 캐시에 존재할 확률이 높고 메인 메모리로부터 다시 정보를 불어오는 일이 적다. 

# 배열의 size 정의와 예외적인 상황
자바에서는 unsigned int가 없어 예외적인 상황이 발생하진 않지만 논리적으로 볼 때 Length가 0일 떄 -1 연산을 수행하는 것은 옳지 않다.  
이는 프로그램이 의도대로 실행되고 있지만 의도한 대로 동작하지 않는다는 것을 의미하는 것으로 주의가 필요하다.
